[Aide Informatique]Le BASH Script !



 
L'extension du BashScript sous Linux est le " .sh " (en réalité, c'est du "Shell" mais le Bash est un interpréteur-compilateur) et sert, tout comme les fichier " .bat " sur Windows, de créer des scripts dans le terminal de Linux !
 
Ces scripts servent souvent à abrégé les lignes de commande à taper vu qu'on les imbriques dans un fichier de script à exécuter !
 
Ainsi, voici les différentes type de lignes que vous pouvez avoir:
 
read variable  => On demande à l'utilisateur du terminal de rentré une donnée qu'on stock dans une variable nommée "variable"
 
echo "vous avez rentré $variable"  => Tout comme avec le PHP, "echo" sert à afficher le texte et nous pouvons remarquer qu'on utilise les variables précédemment déclaré avec le symbole " $ " !
 
for ((ii=0;ii<10;ii++))do echo "Test"; done => Tout comme en Javascript ou PHP (et certainement plein d'autres langage de programmation) on peut créer une boucle avec un début,une condition de fin et une méthode MAIS la particularité de notre Bash est de ne pas utiliser des accolades mais les termes "do" pour débuter et "done" pour fermer !
 
for Test in 'result1' 'result2' 'result3' do echo"$Test "done => Et oui, on peut aussi faire une boucle "for" qui permet de charger une valeur différente à chaque tour dans une variable jusqu'à ce qu'il n'y aille plus de valeur à entrer (dans l'exemple, il y en a 3)
 
if[condition] opérations  fi => Pour les if, c'est la même histoire sauf que cette fois-ci, nullement besoin de mettre "do" avant une opération ! Il suffit juste de préciser la fin du traitement par condition qui est "if" à l'envers !
 
while [condition] do echo"Test" done => Pour les while, c'est la même chose qu'une boucle for mais avec une condition à la manière d'un if... comme dans le Javascript !

Tiens, parlons un peu des comparateurs dans les conditions, contrairement à ce qu'on pourrait croire, nous n'utiliserons pas les symboles de comparateur que les conventions des langages de programmation de logiciels et de pages web dynamique ! 
Les opérateurs sont donc sous forme de fonctions:
 
-eq  => Égale à
 
-ne => Pas Égale à
 
-gt => Plus grand que
 
-ge => Plus grand que ou égale à
 
-lt => Plus petit que
 
-le => Plus petit que  ou égale à
 
-z => Égale à null
 

Puisque ce n'est pas un langage de programmation dépendant d'un serveur (comme PHP) ou de l'HTML/AfterEffect (comme Javascript) mais plutôt du Terminal, vous pouvez scripter des commandes à rentré, comme je l'ai écris plus haut, et donc, pour rappel et pour vous faire baver sur les capacités du Bash, je vous offre un gros récapitulatif des commandes possibles à intégrer dans un script Bash:
https://ss64.com/bash/  (petit lien mais un sacré glossaire complet énorme)
 
 Je vous propose un script simple:
for((ii=0;ii<10;ii++))
do
rm /dossier/fichier$ii
done
 
 
En gros, pour ce script simple, je demande de supprimer les fichiers portant les noms "fichier1","fichier2",... et "fichier10" dans le dossier nommé "dossier" ! L'intérêt de Bash prend tout son sens  dans la gestion des fichiers comme ça !
Malheureusement, ce code supprime définitivement sans demander votre avis ! Donc, il faut ajouter " -i " après rm pour obtenir  une demande de confirmation !
 
 
Lorsque vous tapez beaucoup de code avec des termes de plus en plus complexe, il est normal de commenter son code ! Contrairement au Javascript où on commente après avoir fait " // ", dans le Bash, nous utilisons " # " !
 
Par-contre, pour exécuter un fichier en .sh dans le terminal, il faut l'appeler avec " bash NOMDUFICHIER.sh " !
 
Du coup, pour montrer TOUTE LA CAPACITÉ DE BASH ( MOUAHAHAHAHAHAAAAAAAaaaahh), je vous propose ce code à mettre dans un fichier en .sh :
 
mkdir assets

Ce code permet de créer un site web de base (en gros, les fichiers index.html,style.css et index.js dans les dossiers correspondants ! ) car OUI, on peut non seulement demander la création d'un fichier MAIS ON PEUT AUSSI ÉCRIRE DANS LE FICHIER ET CRÉER DES DOSSIERS !!! (le contrôle est total)

[Aide Informatique] Le MarkDown!

Lorsque vous désirez mettre une description (un README.md) sur Github ou juste personnalisé vos messages sur River.com (et certainement d'autres sites webs), il faut utiliser un langage de balisage (on peut aussi dire "mise en page") spécifique nommé "MarkDown" dont l'extension est le ".md" !
 
Comment on fait ?
Les codes
Lorsqu'on est amené à montrer nos codes HTML,PHP,Javascript,Ruby,Python,... sur des forums ou sur Github, il faut l'insérer entre 3 accents graves ( ``` )!
Mais si on fait juste ça, le code apparaît en noir et blanc donc il faut préciser son langage comme ceci:
```javascript
//Le code en javascript
```
 

Les images

Afin d'intégrer une image dans un document en MarkDown, il faut respecter ce code:
[Nom du fichier]   (AdresseDeL'image)
 
Les liens

Afin d'intégrer un lien dans un document en MarkDown, il faut respecter ce code:
[Texte à afficher](Le lien)

La différence avec l'intégration d'une image est l'espace entre le crochet de fermeture( ] ) et la parenthèse d'ouverture ( ( ) !

Les titres

Pour faire un titre en MarkDown, il y a deux solutions:

1.Le soulignement avec le symbole "="  pour le premier titre, des tirets (-) pour les seconds!
 
Cette méthode est très utilisées dans les ```README.md``` se trouvant dans les dossiers d'installation de logiciel!

2.L'utilisation du symbole "#" auquel il ne faut pas oublier un espace après celui-ci ! On peut aller jusqu'à 5 niveaux de titre différents en rajoutant des dièses!

Les effets dans le texte

italique = *italique*
gras = **gras**
italique gras = ***italique gras***

[Aide Informatique] La fin du "var" !

En Javascript, on a trois moyen de déclarer une variable: var, let et const !
 
Depuis l'ES2015, (l'European Computer Manufacturers Association Script de 2015 , la sixième update), on se permet de mettre en avant const et let pour éviter d'utiliser var!
 
Les différences sont assez simples:
Si on a une variable qui ne change pas, on utilise const!
Si on a une variable qui change, on utilise let!
Si on a un soucis de timing (Temporal Dead Zone) à cause d'une mauvaise position de votre déclaration, on utilise var!
 
Source: https://putaindecode.io/fr/articles/js/es2015/const-let-var/
 
 C'est donc surtout dans un soucis de lisibilité qu'on scinde "var" en "let" et "const" !

[Aide Informatique]Les commandes "git" pour mettre en ligne un projet GitHub!

Lorsqu'on est confronté à l'utilisation de Github sur Linux et Mac, on ne peut utiliser le logiciel créé pour Windows vu que Microsoft a racheté Github en 2018.
 
M'étant confronté à la dur réalité de la chose avec un ordinateur portable fonctionnant avec Ubuntu, j'ai dû utiliser le terminal (Ctrl + Alt + T ) pour pouvoir mettre en ligne des travaux ainsi qu'arranger mes répertoires Github!
 
Pour faire un copier-coller dans le terminal, il faut faire "Ctrl + Maj + C" et "Ctrl + Maj + V", retenez bien ça, c'est pour ne pas copier les URL à la main ou devoir utiliser la souris à chaque fois!
 
Sachez aussi qu'on ne peut pas revenir à une ligne précédente en cliquant dessus!
 
La démarche est simple mais il faut quand même respecter étape par étape la démarche:
 
D'abord, on écrit ces trois lignes de code pour installer Git:
sudo apt-get update
sudo apt-get upgrade
sudo apt-get install git 
 
Vous mettez votre mot de passe
 
 Ensuite, il faut créer un dossier à partir d'un répertoire en ligne sur Github!
Trouver donc un répertoire et cliquer sur "Clone or Download" pour récupérer (copier) un lien avec l'extension .git !

Une fois cela fait, allez dans votre terminal et inscrivez:
git clone https;//github.com/NOM DU PROPRIÉTAIRE DU RÉPERTOIRE/NOM DU RÉPERTOIRE
cd NOM DU RÉPERTOIRE
git branch -a
git branch NOM DE LA NOUVELLE BRANCHE 
git checkout NOM DE LA BRANCHE NOUVELLEMENT CRÉÉE
git branch -a
git branch NOM DE LA BRANCHE DANS LA NOUVELLE BRANCHE
git checkout NOM DE LA BRANCHE DANS LA NOUVELLE BRANCHE
nano README.md
 
Ici, c'est pour écrire en MarkDown la présentation du projet! On continue avec:
 
git add README.md
git commit
 
Ici c'est pour écrire un résumé des modifications apportés au projet! On poursuit avec:
 
git push origin NOM DE LA NOUVELLE BRANCHE
 
Ici, vous inscrivez votre nom d'utilisateur Github et ensuite votre mot de passe!
 
git branch
git push origin NOM DE LA BRANCHE DANS LA NOUVELLE BRANCHE
 
Ici, vous inscrivez votre nom d'utilisateur Github et ensuite votre mot de passe!
Et voila, vous aurez créé une branche et une sous branche que vous aurez mit en ligne!
 
Petit récap' des commandes employées:
 
"git clone " vous permet de copier le répertoire en ligne dans votre ordinateur à l'emplacement dont vous vous trouver actuellement (si vous désirez vous rendre quelque part en particulier, taper juste sa position après la commande "cd")
 
"cd NOM DE DOSSIER" permet de se rendre à un dossier contenu dans le dossier actuel!
 
"cd .." permet de retourner à un dossier en arrière!
 
"git branch X" permet de créer une branche s'appelant X à l'endroit où vous êtes!
 
"git checkout X" permet de mettre sa position à la branche nommée!
 
"nano NOM DE FICHIER.EXTENSION DU FICHIER" permet d'ouvrir le logiciel "Nano" qui sert à écrire dans le terminal ainsi que le fichier qu'on désire modifier!
 
"git add NOM DE FICHIER.EXTENSION DU FICHIER" permet d'ajouter à la liste des trucs à push (mettre en ligne) les fichiers mentionnés!
 
"git commit" permet de faire un rapport avant la mise en ligne!
 
"git push ENDROIT OÙ METTRE LES FICHIERS" permet de mettre en ligne les fichiers qu'on a "git add" ainsi que mettre à jour le "git commit". On vous demande votre nom d'utilisateur et votre mot de passe à GitHub après!
 
 
Voila,je vous ai donc présenté l'essentiel des commandes pour pouvoir utiliser Github sur Linux mais sachez qu'il y a plein de documentation sur le web pour le Mac et l'application pour Windows qui simplifie grandement la tâche!

[Aide informatique]Les notions des fonctions et procédures

Une fonction est un sous-programme constitué de suite de code (instructions) mais qui retourne une valeur.
 
Une procédure ne retournera pas une valeur.
 
Exemple définissant en pseudo-code
 
Fonction RepOuiNon (paramètre)
truc<- " "
Tant que (truc =/= "oui" et truc =/= "non")
Ecrire "taper oui ou non" 
truc<- lire au clavier
fin tant que
Renvoyer truc
Fin
 
Maintenant, à chaque fois que j'écrirais RepOuiNon() dans mon code,je vais utiliser tout le code auquel il est associé à chaque fois.
 
Exemple définissant en javascript

<!Doctype html>
<html>
<head>
<title>Utilisation d'une fonction</title>
<script>
function RepOuiNon(question)
{
alert (question);
var truc="";
while ((truc.toLowerCase()!="oui") && (truc.toLowerCase()!="non"))
truc= prompt ("Répondez par oui ou non");
return truc;
}
</script>
</head>
<body>
<h3>Utilisation d'une fonction</h3>
<script>
var rep1, rep2;
rep1 = RepOuiNon ("Êtes-vous marié?");
rep2 = RepOuiNon ("Avez-vous des enfants?");
document.write ("Vous avez répondu <b>"+rep1+ "</b> à la première question et <b>" +rep2+ "</b> à la seconde question.");
</script>
</body>
</html>
 
 
//1.remarquer qu'on peut mettre autant de balises <script> qu'on veut dans une page html.


//2.remarquer également qu'on emploie truc.toLowerCase() pour que la valeur truc ne prend pas en compte la casse (en gros, peu importe si vous écrivez en majuscule ou en minuscule,la réponse est bonne).


//3.RepOuiNon se remet à jour à chaque réutilisation vu que c'est une fonction et pas une procédure!

[Aide Informatique]Répéter une boucle en javascript pendant un certain nombre de fois!

Lorsque vous créer une boucle en Javascript et que vous désirez embêter en spammant un message, il suffit d'intégrer un compteur.
 
En pseudo-code cela donne:
 
Cpt<- 0
Répéter
 Afficher "message à répéter"
 Cpt<- Cpt+1
jusque (Cpt= le nombre de fois qu'on désire que le message soit répéter)
 
On peut décortiquer le compteur en 3 étapes:
1)Initialisation
2)Incrémentation à chaque fois que l'événement se produit
3)Utilisation


N'oublier pas de faire la légende de votre variable "Cpt"!


Cpt   /   nombre   /   le nombre de "message à répéter" affichés

Par-contre, n'oublier pas que les "jusque" ne sont pas utilisé directement en Javascript,donc on le convertit à l'utilisation du "tant que":

Cpt<- 0
Tant que (Cpt < 10)
Afficher "message à répéter"
Cpt<- Cpt+1
FIn Tant que 

Ce qui donne en javascript:

<script>
var Cpt=0
while (Cpt<nombre de fois que le message à répéter doit être répéter)
{
document.write ("message à répéter <br/>");
Cpt= Cpt+1;
}
</script>

 
Le " <br/> " sert à faire une entré à la ligne!
On aurait pût utiliser " alert " au lieu de " document.write " mais " alert " aurait été chiant à force puisqu'il dira bonjour 10 fois, une fois à chaque clic sur le "ok"!
L'utilisation du " for " aurait pût aussi être utilisé!
for (initialisation, condition de bouclage, incrémentation);
Donc:
<script>
for (Cpt=0; Cpt<nombre de fois que le message à répéter doit être répéter ; Cpt= Cpt+1)
      document.write ("message à répéter");
</script>
On peut abrégé "Cpt=Cpt+1" par "Cpt++" !
